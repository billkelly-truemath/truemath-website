Getting Started

The basics

What TrueMath is, who it's for, and what it costs.

What is TrueMath?

TrueMath is a calculation platform for professionals who need answers they can defend, share, and move on from. You ask a question in plain language, TrueMath figures out which calculation to run, executes it precisely on its own deterministic math engine, and returns the result with full lineage: every variable, every step, every assumption. That lineage is what makes the answer auditable when you're the one being questioned, and what makes it portable when you're the one pitching a client or aligning a counterparty on a deal.

Think of it as the calculation layer that sits between an AI assistant and the math the assistant can't actually do reliably on its own.

Who is TrueMath for?

Professionals whose work depends on getting the numbers right, and on getting the people across the table to agree the numbers are right. Financial advisors walking clients through retirement assumptions. CPAs explaining a return to a partner. M&A advisors quoting fees in an engagement letter. Mortgage brokers showing a borrower how the rate-and-points trade-off actually pencils out. Sales engineers sizing a deal in front of a procurement team.

If you've ever pasted numbers into a chat assistant and then double-checked the answer in Excel because you didn't quite trust it, TrueMath is built for you. If you've ever finished the math and then spent an hour trying to walk a client or a counterparty through how you got there, TrueMath is especially built for you.

What's free, and what costs money?

The Library is free, with a generous monthly allowance of calculations included. You can browse the full set of pre-built calculation domains, run real work in them, and invite colleagues to share an account. Your team works from the same calculation pool.

A paid plan unlocks four things:

  • More monthly calculations than the free tier includes
  • The TrueMath Assistant, which builds new domains for you in plain language
  • Customization: forking and extending Library domains with your firm's specific calculation paths and defaults
  • Sharing beyond your team: sending calculations to clients, counterparties, regulators, or anyone who isn't on your account

During the open beta, everything is free while we tune the product. No credit card required to sign up.

What's a "domain"?

A domain is a curated bundle of calculations for a specific kind of work. The Retirement Income Drawdown Strategy domain, for example, is built for financial advisors planning a client's retirement.

Inside a domain are three things that work together:

Activities are the individual calculations a domain can perform. The Retirement Income Drawdown Strategy domain has activities like Annual Spending Requirement, Bracket-Fill Roth Conversion Amount, IRMAA Surcharge Projection, Sequence-of-Returns Risk Test, and Tax-Efficient Withdrawal Order. Each activity is a defined computation: a series, a single calculation, a simulation, a test, or a decision.

Variables are the typed, self-describing pieces of data that activities consume and produce. A variable like Retirement Age or Taxable Balance carries more than a value. It carries its type (age, currency, percentage), its display units (years, dollars, basis points), its presentation format (a single number, a table, a chart), its decimal precision for display, and an optional default. The underlying value is always stored at full floating-point accuracy; the variable just knows how to present itself sensibly. A variable also carries a description, which is what helps TrueMath map your natural language to the right input.

The connection between them is what makes domains powerful. A single variable typically appears in multiple activities. Retirement Age, for example, is an input to half a dozen activities in a retirement domain. Because activities share variables, they can chain together: the output of one activity becomes the input of another, fully formatted and ready to display, without anyone gluing the pieces together by hand. That chaining is what lets a domain handle questions you didn't pre-author. You're not picking a fixed calculator off a shelf; you're working inside a graph of related calculations that can compose to answer what you actually need.

How It Works

Under the hood

Why the math is reliable and how TrueMath is different from a calculator, a spreadsheet, or a chat assistant.

How is this different from just asking ChatGPT or Claude to do the math?

Large language models are pattern matchers. They predict what numbers should look like, which is why they can confidently produce answers that are almost right but slightly wrong, and they can deliver wrong answers with the same confidence as right ones. There's nothing in the response that tells you which kind you're looking at.

Even when an LLM gets the math approximately right, two runs of the same question can arrive at the same answer through different reasoning paths. Now you have a different kind of audit problem: even if Run A and Run B both produced $4.21M, you can't be sure they got there the same way, with the same assumptions, applied in the same order. For a defensible answer, the path matters as much as the result.

TrueMath uses an LLM only to interpret what you're asking. The math itself runs on a separate deterministic calculation engine built over thirty years by TrueMath co-founder Elia Freedman, the same engine behind PowerOne, which has more than thirty million downloads. The LLM never does the math. Same inputs, same calculation path, same result, every time. That separation is the whole point: a natural language interface, and a calculation you can actually defend.

How is this different from a spreadsheet?

A spreadsheet is a blank canvas. You build the model, you maintain the formulas, you debug the cell references when a colleague's version is slightly different from yours.

TrueMath domains arrive complete. The calculation logic is already there, peer-reviewed and consistent across every user on your team. You bring the inputs and the question; the domain handles the rest.

We watch the Library closely as the world changes, but the more important monitoring is yours. When you customize a Library domain or build a new one with the Assistant, you decide what changes when. The platform makes that easy, and the lineage in every result tells you exactly which version of which activity ran.

What happens when the LLM gets the question wrong?

First, what we mean by "wrong." TrueMath uses an LLM for one specific job: converting your natural-language prompt into the structured request the math engine actually runs (the right activity, the right variables, the right inputs). The math itself never depends on the LLM. So when we talk about LLM errors here, we mean failures in that interpretation step, not in the calculation.

The most common failure looks like this: you describe an input in natural language ("the senior debt rate," say), and the LLM doesn't recognize it as one of the activity's named variables (debt_interest_rate). The activity then has a missing input and can't run, or runs against the wrong variable.

Two things fix this:

  • Improve the variable description. If a variable is described in a way that anticipates the words your team actually uses, the LLM matches it correctly. The Assistant lets you edit variable descriptions in plain language, and that change propagates to every activity that uses the variable.
  • Reword the prompt. The simpler and more direct the request ("Calculate IRR with senior debt at 6.5%"), the more reliably it maps to structured inputs.

Either way, when a calculation fails or looks off, TrueMath shows you exactly what was attempted: which activity, which variables, which inputs. You can read it before you act on it. The math engine doesn't depend on the LLM being right; it just needs to know what to calculate, and you have the final say.

Can the same question return different answers on different runs?

For the same calculation path with the same inputs, no. The math engine is deterministic. Same inputs, same answer, every time.

The interpretation step (the LLM picking which calculation to run) can vary slightly between runs, which is why TrueMath shows you the calculation path it chose. Once you've confirmed the path, the result is reproducible.

Can I build my own domain or extend an existing one?

Yes. The TrueMath Assistant lets you describe what you need to calculate in plain language, and it builds the domain for you. You can also fork an existing Library domain and customize it: add calculation paths specific to your firm, change variable defaults, rename variables to match your terminology.

Building and customizing domains is part of the paid tier. Browsing and using the Library is free.

Trust and Privacy

Where your data goes

How TrueMath is hosted, what happens to your inputs, and how the LLM call is structured.

Where is TrueMath hosted?

Inside AWS. The TrueMath application, the calculation engine, and the LLM interface all run within AWS's secure environment. Data is encrypted in transit and at rest.

Which AI model does TrueMath use?

TrueMath uses Anthropic's Claude, accessed through Amazon Bedrock. The LLM interprets what you're asking and which calculation to run; it does not perform the math itself. The math runs on TrueMath's deterministic calculation engine.

We chose Bedrock specifically because of how it handles data, which is the next question.

What happens to my data when the LLM call is made?

The LLM call goes through Amazon Bedrock, which is AWS's managed service for invoking foundation models. AWS's published Bedrock terms commit to the following:

  • Customer prompts and completions are not used to train the underlying foundation models
  • Model providers, including Anthropic, do not have access to your prompts or completions
  • Data stays inside AWS infrastructure during the call and is not exposed to the public internet
  • Inputs and outputs are encrypted in transit and at rest

TrueMath operates within these terms. Your calculation inputs, the prompts derived from them, and the LLM's responses stay inside the AWS environment for the duration of the request.

For the most current details, AWS publishes the Bedrock data privacy commitments at aws.amazon.com/bedrock/security-compliance.

Does TrueMath use my data to train any models?

No. TrueMath does not train models on your calculation inputs or outputs, and the underlying foundation models accessed through Bedrock are not trained on your prompts either.

Is TrueMath multi-tenant or single-tenant?

TrueMath is a multi-tenant application with standard AWS-grade tenant isolation. Each customer's data is logically separated, encrypted, and accessed only through authenticated, authorized API calls scoped to that customer.

Enterprise customers with stricter requirements should contact us; we're happy to discuss what's possible for specific compliance regimes.

Can someone outside my team see a calculation if I share it with them?

Yes, when you're on a paid plan. Sharing a calculation outside your team produces a viewable record with full lineage: every variable, every activity, every assumption. The recipient can examine it without needing a TrueMath account. They see exactly what you saw.

The shared record is a snapshot of that specific calculation. It's not a window into your account, your team's other work, or any inputs you didn't deliberately include. The recipient can read; they can't run new calculations on your dime or see anything else.

Sharing within your team is included in every plan, free or paid.

Is the math itself audit-ready?

Every TrueMath result includes the calculation path used, the variables and values applied, and the assumptions invoked. You can show the lineage to a client, a counterparty, an auditor, or a regulator and walk them through every step. The same record that lets you defend the answer internally is what lets you align stakeholders externally.

TrueMath does not produce "estimates" with hidden assumptions. If something was approximated, the approximation is in the lineage.

For Developers

API and integration

Embedding TrueMath into your own product, agent, or workflow.

Does TrueMath have an API?

Yes. The TrueMath REST API is available now. Send a domain identifier and either a natural-language prompt or a structured prompt with named variables; get back a precise, audit-ready answer with full calculation lineage.

SDKs for Python and JavaScript/TypeScript, plus an MCP integration for agentic workflows, are in development.

How do I get API access?

Sign up for an account and request API access from your dashboard, or contact us directly. During beta, API access is granted on a case-by-case basis so we can support each integration well.

What does a typical API request look like?

You hit POST /v1/calculate with a JSON body containing the domain identifier (e.g. "cre_investor") and either a natural-language prompt or a structured prompt with named variables. You don't need to send both. If you send natural language, the LLM converts it to structure on the way in. If you send structure, the call goes straight to the math engine.

You get back the result, the calculation path, and the lineage. The homepage Developers section shows the request and response shapes.

Can I use TrueMath as a tool inside an AI agent?

That's one of the primary use cases. If you're building an agent that needs to do real math, TrueMath is the calculation layer the agent calls when it hits a number it actually has to get right. The agent decides what to calculate; TrueMath calculates it deterministically and returns a defensible answer.

An MCP integration is in development to make this even more direct.

What about pricing for API usage?

API pricing tiers will be published when we exit beta. During beta, API access is free for approved integration partners. If you're building something specific, get in touch and we'll discuss what makes sense.

Is TrueMath available on AWS Marketplace?

Not yet, but soon. We're in the process of listing TrueMath on AWS Marketplace, which will make procurement easier for enterprise customers who already have AWS spend committed. The Marketplace listing will cover the same product available at app.truemath.ai. Get in touch if you'd like to be notified when it goes live.

How it works

One request. One certain answer.

Send a natural language prompt and your input values. TrueMath parses the intent, routes to the correct calculation, executes the math precisely, and returns a structured, auditable result.

01 Send your prompt

POST a natural language question and your input values as JSON. No special syntax or formula knowledge required.

02 TrueMath routes and calculates

The engine parses the prompt, identifies variables and the target output, selects the correct calculation path, and executes deterministically.

03 Receive an auditable answer

The response includes the result, the calculation steps taken, the variables used, and the domain and activity that produced the answer.

Code example

Simple request. Structured response.

A single POST to /v1/calculate returns a complete, auditable answer with full calculation lineage.

The domain field scopes the calculation to a specific role. Omit it and TrueMath will infer the best match from your prompt.

REQUEST
// POST /v1/calculate
{
  "domain": "cre_investor",
  "prompt": "What is the IRR on
    this deal?",
  "inputs": {
    "purchase_price": 4200000,
    "noi": 273000,
    "hold_period_years": 5,
    "exit_cap_rate": 0.065
  }
}
RESPONSE
{
  "result": {
    "irr": 0.1243,
    "irr_formatted": "12.43%"
  },
  "domain": "cre_investor",
  "activity": "levered_irr",
  "steps": [...],
  "inputs_used": {...},
  "deterministic": true
}
API Reference

Endpoints

A focused set of endpoints covering calculation, domain management, and session context. Full reference documentation coming soon.

POST
/v1/calculate
Submit a natural language prompt with input values. Returns a deterministic, auditable answer with full calculation lineage.
GET
/v1/domains
List all domains available to your account, including Library domains and any custom domains you have built.
GET
/v1/domains/{domain_id}/activities
List all activities within a domain, including variable descriptions and supported solve directions.
POST
/v1/sessions
Create a calculation session. Variables set in a session are carried forward automatically across subsequent requests.
POST
/v1/sessions/{session_id}/calculate
Submit a calculation within an existing session. Prior variable values are available without re-supplying them.
